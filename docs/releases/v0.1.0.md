# GS Mapper v0.1.0 Release Notes

This release packages GS Mapper as a practical bridge from photos, robotics
logs, and external SLAM outputs to trainable 3D Gaussian Splatting artifacts
that can be opened in a browser.

## Highlights

- One-shot `gs-mapper photos-to-splat` flow for image folders.
- Bundled GitHub Pages viewers for point bundles, WebGL `.splat`, Spark, and
  WebGPU rendering.
- Eight production demo scenes comparing supervised GNSS + LiDAR, DUSt3R,
  MASt3R, MASt3R-SLAM, and VGGT-SLAM outputs.
- External SLAM import path for MASt3R-SLAM, VGGT-SLAM 2.0, Pi3 / Pi3X, LoGeR,
  and generic TUM/KITTI/NMEA pose outputs.
- Dry-run external SLAM preflight manifests with gate failures, selected
  artifact paths, image-directory status, and candidate-resolution traces.
- Pi3 / LoGeR tensor container import for `.npy`, `.npz`, `.pt`, and `.pth`
  pose or dense point outputs.

## Quickstart

```bash
git clone https://github.com/rsasaki0109/gs-mapper.git
cd gs-mapper
pip install -e ".[dev]"

gs-mapper photos-to-splat --images ./my_photos --output outputs/my_splat
```

The command writes a `.splat` file that can be opened through
`docs/splat.html?url=<path-to-splat>`.

## External SLAM Preflight

```bash
python3 scripts/plan_external_slam_imports.py --format shell > /tmp/external_slam_preflight.sh
bash /tmp/external_slam_preflight.sh
python3 scripts/collect_external_slam_imports.py --format markdown
```

Use this before training to confirm that external SLAM artifacts are present,
selectable, and aligned with the expected image directory.

## Validation

Release-facing validation run:

```bash
python3 -m pytest tests/test_external_slam.py tests/test_external_slam_import_plan.py tests/test_cli.py -q
```

Current result: `118 passed, 17 warnings`.

## Known Limits

- DUSt3R and MASt3R upstream models carry non-commercial license constraints.
- Heavy external SLAM dependencies are intentionally kept outside this repo;
  GS Mapper imports their exported artifacts as data.
- Some bundled demo scenes are comparison-quality rather than default-quality
  production outputs, and their README rows call this out explicitly.
